# Trademark Policy

This trademark policy governs the use of the "Dazi" and "搭子" names, logos, and branding in connection with the Dazi open source project.

## Trademarks

The following are trademarks used in this project:

- **Dazi** (word mark)
- **搭子** (word mark)
- The Dazi logo and icon

## What You Can Do

### Use the Code Freely

The Dazi source code is licensed under the Apache License 2.0. You are free to:

- Use, modify, and distribute the code
- Create derivative works
- Use the software for any purpose, including commercial use

### Make Factual Statements

You may make accurate, factual statements about your relationship to the project:

- "Based on Dazi"
- "Built with Dazi technology"
- "Compatible with Dazi"
- "Fork of Dazi"

### Contribute to the Project

Contributors to the official Dazi repository may use the trademarks when discussing their contributions.

## What You Cannot Do

### Use Dazi Branding for Forks

If you create a fork or derivative work, you **must**:

- Choose a different name that does not include "Dazi" or "搭子"
- Remove or replace all Dazi logos and icons
- Update the bundle identifier (e.g., `com.example.dazi`) to your own
- Remove references to project-controlled domains unless connecting to official services

### Imply Official Endorsement

You may not:

- Use "Dazi" or "搭子" as your product name
- Use the Dazi logo as your application icon
- Suggest that your fork is the official version
- Imply that the Dazi project endorses your product

### Create Confusion

You may not use the trademarks in any way that:

- Suggests your product is created by or affiliated with the Dazi project
- Could cause confusion between your product and the official Dazi project
- Disparages the Dazi project

## Branding Locations

For those creating forks, the following files contain branding that should be updated:

| File | Contains |
|------|----------|
| `apps/electron/electron-builder.yml` | Product name, bundle ID, copyright |
| `apps/electron/resources/` | Application icons |
| `packages/shared/src/config/branding.ts` | Service URLs |

## Examples

### Acceptable

- "MyAgent - based on Dazi"
- "This project is a fork of Dazi"
- "Compatible with the Dazi ecosystem"

### Not Acceptable

- "Dazi Pro"
- "搭子 for Linux"
- "Better Dazi"
- Using the Dazi logo for your fork
